# 🏢 Office TD — Enikey Defense

**Tower Defense** в офисном сеттинге. Петя сидит в своём кабинете, а по коридору к нему идут коллеги с тасками. Помоги Пете выжить!
////////
## 🎮 Стек

| Технология | Версия |
|---|---|
| Phaser 3 | ^3.88 |
| TypeScript | ~5.8 |
| Vite | ^6 |
| Node.js (CI) | 22 LTS |

## 🚀 Локальный запуск

```bash
npm install
npm run dev        # http://localhost:5173
```

## 🔧 Сборка

```bash
npm run build          # web-версия → dist/
npm run build:electron # Electron-версия → dist/ (base='./')
npm run build:cap      # Capacitor-версия → dist/ (base='./')
```

## 📦 Платформенные пакеты

```bash
# Windows: .exe инсталлятор (нужен Node + electron-builder)
npm run dist:win       # → release/*.exe

# Android: инициализация (первый раз)
npx cap add android
npm run cap:sync       # vite build + cap sync → android/
npm run cap:open       # открыть в Android Studio
```

Экран портретный (480×800, под мобилки) — коллеги заходят через двери в
верхней четверти (ресепшен) и идут вниз через кабинет Пети (нижние 3/4
экрана) к его столу. Кабинет — это и есть игровое поле для построек.

## 💰 Экономика

Петя стартует с деньгами (`STARTING_MONEY`) и получает ещё немного
(`WAVE_MONEY_AMOUNT`) за каждую полностью зачищенную волну. За деньги можно
ставить новые башни или улучшать существующие (клик по башне, до 3 уровней —
больше урон, дальше радиус, быстрее стрельба).

Волна 1 начинается сразу. Дальше — цикл: волна идёт, пока не будут
обработаны (убиты или дошли до стола) все её коллеги, затем начисляются
деньги и наступает **пауза** — можно спокойно тратить накопленное на
постройку и апгрейды, без давления таймера. Следующая волна не начнётся,
пока не тапнешь по кнопке ▶️ **«Начать волну»** (под кнопками способностей,
в правом верхнем углу — появляется только на время паузы).

## 📥 Очередь задач (Inbox)

Коллега, дошедший до стола, больше не отнимает у Пети «жизнь» напрямую —
он кидает свою задачу в **Inbox** (`src/systems/Inbox.ts`), очередь с
лимитом (`INBOX_LIMIT`, по умолчанию 10). Превысил лимит — Game Over.

Петя **саморешает** задачи: каждые `INBOX_RESOLVE_INTERVAL_MS` (2с) он
автоматически снимает задачу с начала очереди — это происходит само,
без участия игрока. Коллеги подходят к столу быстрее, чем Петя успевает
разгребать, так что очередь копится — единственный способ сдержать наплыв
это выбивать коллег башнями по пути (📜 Script, 📖 Docs и остальные), пока
они ещё идут по коридору, не дав им вообще добраться до стола.

Часть коллег (`URGENT_TASK_CHANCE`, 15%) несёт **срочную задачу** 🔴 —
такая, попав в Inbox, встаёт не в конец очереди, а **в начало** (обгоняет
всех) — её тикет на враге тоже подсвечен красным, чтобы такого коллегу
было видно заранее и приоритетно снять его башнями. В HUD под счётчиком
Inbox — мини-ряд из `INBOX_LIMIT` значков, показывающий очередь как есть
слева направо (левый — следующий на решение): 🔴 срочная, 🟡 обычная,
▫️ свободный слот.

## 🎫 Способности

Две заряжаемые способности, кнопки — в правом верхнем углу (кольцо вокруг
кнопки показывает прогресс зарядки; когда полная — кнопка пульсирует).

**🎫 «Создай тикет»** — глобальный удар, откат 30с (`ULTIMATE_COOLDOWN_MS`).
По тапу половина живых коллег на экране (округление вверх,
`ULTIMATE_KILL_FRACTION`) мгновенно «сдувается» и уходит — им лень заводить
официальную заявку в Jira. Считается как обычное устранение для прогресса
волны — так же приближает её зачистку и следующую паузу.

**🛡️ «Я на митинге»** — временная неуязвимость двери, откат 25с
(`SHIELD_COOLDOWN_MS`), длительность 5с (`SHIELD_DURATION_MS`). Пока щит
активен, любой дошедший до стола коллега вместо удара по Пете застревает
у таблички «Не беспокоить» и получает периодический урон от ожидания
(`SHIELD_DOT_DAMAGE` каждые `SHIELD_DOT_INTERVAL_MS`) — башни тоже продолжают
по нему бить. Если не добить за время действия щита — по истечении коллега
всё же врывается и засчитывается как обычно.

## 🪑 Мебель

Кроме башен, в кабинет можно ставить мебель — она бесплатная, но у каждого
типа конечный запас (`FURNITURE_TYPES_DATA`, `src/config.ts`). Мебель не
атакует, это чистое препятствие на пути коллег:

| Мебель | Запас | Особенность |
|---|---|---|
| 🗄️ Шкаф | 3 | Непроходим — коллеги огибают его |
| 🗃️ Тумбочка | 4 | Непроходима, но компактнее шкафа |
| 🛋️ Диван | 2 | Непроходим; коллега, дошедший до него, останавливается и садится на `SOFA_SIT_DURATION_MS` (3с) — башни всё это время продолжают по нему бить — прежде чем встать и пойти дальше |

Мебель делит общий цикл выбора построек с башнями (клавиши 1-9, правый
клик, тап по строке типа в HUD — сначала идут 6 башен, потом 3 предмета
мебели). Уже поставленную мебель можно **передвинуть**: тап по ней
поднимает её (следует за курсором), следующий тап по свободному месту в
кабинете кладёт её туда; тап по невалидному месту возвращает мебель туда,
где она стояла.

## 🕹️ Управление

| Действие | Управление |
|---|---|
| Поставить башню/мебель | Клик/тап по кабинету |
| Улучшить башню | Клик/тап по уже стоящей башне |
| Переставить мебель | Клик/тап по ней (взять), клик/тап по новому месту (положить) |
| Сменить тип постройки | Клавиши 1-9, правый клик, или тап по строке типа в HUD |
| Посмотреть радиус башни | Навести курсор / тап |
| Активировать «Создай тикет» | Тап по кнопке 🎫 (когда заряжена) |
| Активировать «Я на митинге» | Тап по кнопке 🛡️ (когда заряжена) |
| Начать следующую волну | Тап по кнопке ▶️ (появляется в паузе между волнами) |
| Рестарт (после Game Over) | **R** |

### Типы башен

Характеристики и стоимость заданы в `TOWER_VARIANTS_DATA` (`src/config.ts`).

| Башня | Особенность |
|---|---|
| 📜 **Script** («Автоматизация») | Базовая скорострельная, стреляет зелёными «0»/«1» (матрица). Убийство рикошетит в ближайшего другого врага в радиусе за половину урона — и так по цепочке, пока рикошет добивает (до `CHAIN_MAX_BOUNCES` прыжков). Отлично чистит толпу слабых врагов |
| 📡 **Router** («Плохой коннект») | Чистый дебаффер: без снаряда — кольцевая Wi-Fi волна от самой башни, замедляет всех в радиусе («зависают, грузя страницу»). Урона по умолчанию нет (только апгрейды дают немного) — держит толпу под другими башнями |
| 📖 **Docs** («Бюрократия») | Медленная тяжёлая артиллерия — швыряет «талмуды» 📖 (RTFM), только по одной цели, колоссальный урон + стан. Помечена как игнорирующая броню (`armorPiercing`) — задел на будущее: пока в игре нет врагов с бронёй, флаг ни на что не влияет |
| ☕ **Coffee** | Замедляет цель при попадании |
| 🪑 **Chair** | Дешёвая стартовая башня (слабая, но доступна с самого начала) |
| 🌬️ **AC** | Урон по области вокруг попадания |

Все башни прокачиваются кликом/тапом по уже поставленной (до 3 уровней —
сильнее урон, больше радиус, быстрее стрельба; растёт и стоимость апгрейда).

## 🖼️ Текстуры

Сейчас всё рисуется программно (`Graphics`/фигуры/эмодзи), без картинок —
но слоты под полноценную графику уже заведены на **всё**: пол кабинета,
пол коридора, стену, дверь, стол (с Петей), всех 6 башен, коллегу-врага и
все 3 предмета мебели.

**Полное ТЗ для генерации — [`public/assets/ART_BRIEF.md`](public/assets/ART_BRIEF.md)**:
стиль, палитра (взята из уже нарисованных программно цветов), точные
размеры и что должно быть на каждой картинке — документ рассчитан на то,
чтобы отдать его напрямую ИИ-агенту, генерирующему изображения, или
художнику.

Механика подключения простая: кладёте PNG с нужным именем в
`public/assets/tiles/` или `public/assets/sprites/` (имена и пути — в
`TEXTURE_ASSETS`, `src/config.ts`) — `MainScene.preload()` подхватывает их
автоматически, код трогать не нужно. Пока файла нет, соответствующий
элемент рисуется как сейчас (программная заглушка), так что игра работает
в любом промежуточном состоянии — можно доливать ассеты по одному.

## 🔄 CI/CD (Jenkins)

Пайплайн описан в [`Jenkinsfile`](./Jenkinsfile).

### Этапы

```
Source Checkout       (+ чистка dist/release/android от прошлой сборки)
    ↓
Build Node Image      (пересобирается и пушится, только если изменился Dockerfile.build)
    ↓
Build Android Image   (аналогично — только при изменении Dockerfile.android)
    ↓
Install Dependencies  (npm ci, с постоянным кэшем — том itdefence-npm-cache)
    ↓
TypeScript Check      (tsc --noEmit)
    ↓
┌───────────────────────────────────────┐  ← последовательно (ARM-хост не тянет parallel)
│ Build: Web           │ Build: Android          │
│ vite build:web       │ Capacitor + Gradle      │
│ → dist/ → Nginx-образ │ → android/...apk        │
│                       │ (кэш: itdefence-gradle-cache) │
└───────────────────────────────────────┘
    ↓
Archive (dist/**, **/*.apk)
    ↓
Deploy Web [только main] → docker compose (ssh) → /opt/itdefence на 192.168.10.222:7979
```

> **Windows .exe не собирается в этом пайплайне.** `electron-builder --win` создаёт
> NSIS-инсталлятор, а это требует Wine для запуска Windows-бинарников — на ARM64
> Linux-хосте Wine работал бы только через x86-эмуляцию (box64/qemu), что
> ненадёжно и очень медленно. Собирайте Windows-версию (`npm run dist:win`)
> либо локально на Windows, либо на отдельном Windows-агенте/раннере.

> **Android-сборка использует сторонний `aapt2` для arm64.** Google публикует
> `aapt2` только под x86_64/macOS/Windows, а Android Gradle Plugin тянет
> именно эту версию с Maven независимо от SDK build-tools — на ARM-хосте
> она не запускается. `Dockerfile.android` подкладывает вместо неё бинарник
> из [Commit451/android-arm-build-tools](https://github.com/Commit451/android-arm-build-tools)
> (MIT, пересобран из тегов релиза AOSP), запиненный на конкретный релиз и
> проверенный по SHA-256 при сборке образа. Если после апдейта AGP снова
> сломается (несовместимый флаг aapt2) — нужно поднять версию релиза в
> `Dockerfile.android` и обновить хэш.

> **JDK берётся не из apt, а из тарбола Eclipse Temurin.** Capacitor 7
> генерирует Android-проект под Java 21, а Debian bookworm пакует только
> openjdk-17 (openjdk-21 появился лишь в Debian 13). `Dockerfile.android`
> ставит [Temurin 21 arm64](https://github.com/adoptium/temurin21-binaries)
> из checksum-пиненного тарбола вместо apt-пакета.

### Необходимые секреты Jenkins

| ID | Тип | Описание |
|---|---|---|
| `serge` | SSH Username with Private Key | Ключ пользователя `serge` на целевом сервере (deploy-пользователя `deploy` на хосте нет — используется тот же credential/юзер, что и в других проектах на этом Jenkins) |

### Параметры сборки

| Параметр | По умолчанию | Описание |
|---|---|---|
| `SKIP_TYPECHECK` | false | Пропустить tsc-проверку |
| `BUILD_WEB` | true | Собирать веб-версию и деплоить на сервер |
| `BUILD_ANDROID` | true | Собирать Android .apk |
| `FORCE_DEPLOY` | false | Деплоить не из ветки main |
| `FORCE_REBUILD_IMAGES` | false | Пересобрать Node/Android toolchain-образы, даже если их Dockerfile не менялся |
