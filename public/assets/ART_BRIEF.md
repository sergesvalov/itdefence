# Office TD — арт-бриф для генерации текстур

Это техническое задание для ИИ-агента (или художника), который будет
генерировать графику для игры. Игра сейчас полностью работает на
программной графике Phaser (`Graphics`/фигуры/эмодзи) — она собой не
блокирует ничего и продолжит работать, даже если часть ассетов ниже
никогда не появится. Цель этого документа — дать достаточно контекста,
чтобы сгенерированные картинки выглядели так, будто их рисовали одним
человеком за один присест, а не набор случайных стилей.

## Как это подключается (коротко)

`src/config.ts` → `TEXTURE_ASSETS` — единый список: ключ, путь, тип
(`tile` — повторяющаяся текстура пола/стены, `sprite` — одна картинка).
`MainScene.preload()` пытается загрузить все файлы по этим путям; если
файла нет — 404 тихо игнорируется, и соответствующий элемент карты/сущность
рисуется как сейчас (программной графикой). **Никакого кода трогать не
нужно** — просто положить PNG с правильным именем в правильную папку.

## Стиль и настроение

- **Плоская векторная мультяшность**, НЕ пиксель-арт, НЕ фотореализм,
  НЕ отрисовка вручную/акварель. Простые формы, мягкие скругления, минимум
  градиентов (лёгкий градиент/тень для объёма — ок, сложные текстуры
  материалов — нет).
- Сеттинг: офисная сатира. Tower Defense, где Петя обороняет свой стол от
  коллег, несущих таски/тикеты. Тон лёгкий, ироничный, не мрачный.
  Референсы по духу: корпоративные мемы, флэт-иллюстрации в стиле
  "офисная жизнь", ничего страшного/жестокого.
- Ракурс:
  - **Тайлы полов (`tile-*`)**: Строгий вид сверху (strict top-down), без искажений и изометрии. Это плоская основа.
  - **Спрайты объектов и персонажей (`sprite-*`)**: **Top-down 3/4** (он же *RPG top-down* или *High-angle 3/4 top-down view*). Объекты стоят на плоском полу, но мы видим их под углом (примерно 45-70 градусов сверху). Из-за этого видна макушка и передний фасад, что придаёт плоским спрайтам объём. Это **не изометрия** (сетка пола не повернута ромбом).
- Обводка/контур: сейчас все фигуры имеют чёткую контрастную обводку
  (2px, на 1-2 тона темнее заливки) — сохраняйте этот приём, он держит
  силуэт читаемым на пёстром фоне.

## Технические требования

- Формат: **PNG с прозрачным фоном** (для `sprite`-типа обязательно;
  для `tile` фон непрозрачный, т.к. текстура заливает area целиком).
- Тайлы (`tile-*`) должны **бесшовно повторяться** по обеим осям — края
  должны совпадать при стыковке друг с другом.
- Спрайты (`sprite-*`) рендерятся через `setDisplaySize()` под
  фиксированный размер независимо от реального разрешения файла — но
  сохраняйте пропорции, указанные в таблице, иначе будет растяжение.
  Компонуйте с отступом от края кадра (не впритык), иначе при масштабировании
  обрежутся углы.
- Держите файлы лёгкими (это мобильная игра): тайлы — единицы-десятки КБ,
  спрайты — до ~100-150 КБ каждый. Не нужно 4K-исходники.
- Экспортируйте сразу в целевом разрешении из таблицы ниже (или кратном
  ему — 2x для ретины — если генератор это умеет; код всё равно
  масштабирует через `setDisplaySize`).

## Приоритет (если генерировать не всё сразу)

1. `desk.png` (стол с Петей) и `coworker.png` — самые заметные, на экране
   почти всегда.
2. `tile-office-floor` + `tile-corridor-floor` + `tile-wall` — задают
   общее настроение уровня.
3. `door.png` — заметен, но маленький.
4. 6 спрайтов башен — можно доливать по одному, старая иконка-заглушка
   для остальных вариантов не мешает вперемешку со спрайтами уже готовых.
5. 3 спрайта мебели (`furniture-cabinet`, `furniture-drawer`,
   `furniture-sofa`) — самый низкий приоритет, тот же принцип постепенной
   доливки.
